# Implement from SpecForge bundle

## Preconditions

- `spec.yaml` exists and is the source of truth.
- Supporting files present as generated by SpecForge: `CLAUDE.md`, `acceptance-checklist.md`, and optionally `.claude/commands/implement-from-spec.md`.

## Steps

1. Read `spec.yaml` (objective, constraints, hypotheses, actions, success criteria, required evidence).
2. Follow actions in a sensible order; honor `supports` links to hypotheses.
3. Satisfy `required_evidence` with concrete artifacts (test runs, command output).
4. Complete `acceptance-checklist.md` honestly.
5. Update `CLAUDE.md` only with stable decisions and pitfalls—not scratch logs.

## MCPs

Use MCPs to verify integrations, run tests, or fetch live data needed for evidence. Do not expand product scope beyond the spec unless the user explicitly revises `spec.yaml`.
